# 周报、清单与可视化设计

## 目标

让输出同时满足两件事：

- 第一眼就觉得专业、锐利、值得继续看
- 第二眼就能直接转成行动

所以设计原则是：

`地图和图表负责抓眼球，清单和结论负责成交。`

## 视觉方向

### 风格关键词

- 都市感
- 政策雷达感
- 冷静、专业、略带交易桌面风格
- 不做花哨地产 brochure 风

### 建议配色

- 背景：`#0B1020`
- 主高亮：`#33D1FF`
- 机会高亮：`#B7FF3C`
- 提醒高亮：`#FF5DAA`
- 风险提示：`#FFB547`
- 阻塞/剔除：`#FF5C5C`
- 中性线条：`#5D6B8A`

### 字体建议

- 正文：`Inter` 或 `IBM Plex Sans`
- 数字和标签：`JetBrains Mono` 或 `IBM Plex Mono`

### 设计规则

- 一个页面只强调一个主结论
- 地图上不要堆太多图层文字
- proposal 阶段颜色要全产品统一
- 让 `Top N` 与 `High Priority` 视觉上明显高于普通 watchlist

## 最值得做的 5 类可视化

## 1. Opportunity Map

最适合首页、周报封面和演示。

地图上建议显示：

- 重点 station / centre
- TOD / LMR 覆盖区
- proposal 点位或 precinct
- 高机会条目
- 主要约束边界的简化轮廓

视觉原则：

- 深色底图
- 机会点用高亮荧光色
- 阶段不同用颜色区分，不靠形状区分全部含义
- 高机会点允许带 glow 效果，但数量必须少

它回答的问题是：

`本周哪里在发热？`

## 2. Policy Pipeline Chart

建议做成阶段漏斗或阶段堆叠图。

可展示：

- `Under Assessment`
- `Pre-Exhibition`
- `On Exhibition`
- `Post-Exhibition`
- `Finalisation`
- `Made`

也可以加一条最近 12 周的变动趋势线。

它回答的问题是：

`机会是在累积，还是在兑现？`

## 3. LGA Scoreboard

适合排序和对比。

建议字段：

- 本周新增 proposal 数
- 本周状态升级数
- TOD / LMR 命中 precinct 数
- 住房目标压力信号
- 风险调整后机会数

最适合做成：

- 横向条形排名图
- 简洁热力表

它回答的问题是：

`本周最值得盯的区是哪几个？`

## 4. Precinct Story Map

最适合深度分析。

同一页里建议放：

- precinct 边界地图
- 800m 覆盖圈或步行覆盖概念层
- proposal / DA / infrastructure 标记
- 主要 constraints
- 旁边放 4-6 条分析结论

它回答的问题是：

`这个片区为什么值得看，具体值在哪里？`

## 5. Site Card Grid

最适合 `Top N` 清单。

每张卡片只放：

- 名称 / 地址 / precinct
- 触发原因
- 机会评级
- 风险标签
- 推荐动作

它回答的问题是：

`我现在先看哪几个？`

## 6. Attention Trend Chart（实验性）

只有在舆情层能稳定采集后再做。

建议展示：

- suburb / precinct 讨论热度趋势
- 热度变化与 policy event 的时间对照
- controversy 高点与 planning progress 的关系

它回答的问题是：

`市场开始讨论了吗，还是仍停留在政策早期阶段？`

注意：

- 不建议把它做成“单一情绪仪表盘”
- 更适合做 `attention` 和 `narrative shift` 图，而不是纯 sentiment 分数图

## 周报模板

## 首页结构

1. 本周 headline
2. 本周三大结论
3. Opportunity Map
4. Top 10 Hotlist
5. LGA Scoreboard
6. Key policy changes
7. Next to watch

## 周报 headline 写法

好的标题应该像交易台晨报，不像地产广告。

示例：

- `Parramatta-Ryde corridor leads this week’s policy acceleration`
- `Three station precincts moved into higher-conviction territory this week`
- `Canterbury-Bankstown shows the strongest watchlist expansion under current policy settings`

## 周报中的三类结论

每期固定写这三句：

- `What changed`
- `Why it matters`
- `What to do next`

## Top N 清单模板

建议字段：

| 字段 | 说明 |
| --- | --- |
| Rank | 排名 |
| Item | 地址或 precinct 名称 |
| LGA | 所属 council |
| Trigger | 本次入选的主要原因 |
| Catalyst | TOD / LMR / proposal / infrastructure |
| Friction | 主要约束 |
| Rating | A / B / C |
| Action | Monitor / Investigate / Deep dive / Contact owners |

### Top N 的选入规则

- 一定要有明确触发事件
- 一定要能说明“为什么是现在”
- 一定要有推荐动作
- 没有动作建议的条目，不应进入 Top N

## Alert 模板

高优先级 alert 不要长。

固定结构：

### 标题

`[High Priority] <precinct or site> moved into <new stage>`

### 正文四行

- 发生了什么
- 为什么重要
- 主要风险
- 建议下一步

## Deep Dive Memo 模板

建议控制在 `8-12` 页或等价网页长度。

### 建议章节

1. Executive Summary
2. Investment / acquisition thesis
3. Policy and planning context
4. Precinct map and boundary logic
5. What has changed in the last 3-12 months
6. Supply context and DA pattern
7. Key constraints and execution friction
8. Best site archetypes
9. Ranked watchlist
10. Recommended actions in the next 30-90 days

## Site Card 模板

每个推荐条目都应该能独立截图传播。

### 标准结构

- 名称
- 所在 precinct / LGA
- 这次为什么浮上来
- 主要政策催化
- 主要约束
- 置信度
- 推荐动作

### 推荐动作标签

- `Watch`
- `Investigate`
- `Prioritise`
- `Drop`

## 讲故事的顺序

无论周报、Top N 还是 Deep Dive，都按这个顺序讲：

1. 先说结论
2. 再说发生了什么变化
3. 再说这对开发能力和 timing 的影响
4. 最后说风险和下一步动作

## 不该出现的表达

- `这是一个绝佳投资机会`
- `一定会涨`
- `被严重低估`
- `AI 预测未来价格`

更安全、更专业的表达：

- `policy-led opportunity`
- `higher-conviction watchlist item`
- `screening candidate`
- `requires deeper due diligence`
- `timing appears to be improving`

## 一页式线框建议

```text
+--------------------------------------------------------------+
| Headline                                                     |
| 1-2 句本周核心判断                                           |
+-------------------------------+------------------------------+
| Opportunity Map               | Top 5 Items                  |
| 高亮站点、precinct、proposal   | 排名、触发原因、动作建议      |
+-------------------------------+------------------------------+
| Policy Pipeline               | LGA Scoreboard               |
| 阶段漏斗/趋势                  | 本周区域排序                  |
+--------------------------------------------------------------+
| What changed / Why it matters / What next                    |
+--------------------------------------------------------------+
```

## 最终建议

先把视觉资源用在这三类页面：

- 周报首页
- Top N shortlist 页面
- 单 precinct 深度分析页

因为这三类页面最容易同时承担：

- 演示
- 客户转化
- 日常使用
- 品牌感建立

如果后面要扩展 dashboard，也应先从这三页衍生，而不是反过来从复杂系统里拆页面。
