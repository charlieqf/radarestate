示例基准：当前正式周快照为 2026-04-05。当前客户包进一步展开为 1 份 Full Report、1 份 Delta Report、1 份 Methodology Appendix、1 份 Top 10 Insights、1 份 Weekly Radar、1 份 Top Site Screening、12 份 Site Cards、4 份 Precinct Deep Dives、1 份 Hero Visual Pack、1 份 Opportunity Map 与分组后的客户交付门户。Sydney 的 housing target context 在当前产品定义的 target-council scope 内覆盖 22/22 个 council。

1. 数据源的广泛性与权威性

公开来源 + 结构化治理 + 时间范围留痕
数据层 具体来源 在流程中的作用 当前可见覆盖例子
NSW Planning Portal
Planning Proposals		 来自 NSW Planning Portal 的 planning proposal 公示页面与 proposal 详情页。客户可以把它理解成“新州规划系统里公开披露的 proposal 名单、阶段和对应说明页面”。 形成 precinct 级政策信号、proposal watchlist、stage pipeline、policy score 当前正式快照记录了 proposal 源时间范围，且会进入 stage history 与周快照归档
NSW Planning Portal
DA Application Tracker		 来自 NSW Planning Portal 的开发申请跟踪数据。客户可以把它理解成“各 council 公开开发申请记录的统一抓取层”，其中包含 DA、CDC、Modification 等常见申请类型。 形成 DA / CDC / Modification / Other 的 application signal 基础层，以及 timing score 和活跃度判断 当前源层累计 application signals 为 45,114；客户口径下自 2025-01-01 起累计为 43,191
NSW Planning Portal
State Significant Projects		 来自 NSW Planning Portal 的 state significant projects 公共项目数据。客户可以把它理解成“新州政府直接评审或具有州级意义的重大项目名录与空间位置”，不是一般 council DA。 补充 state-significant 项目对 timing 与 activity 的影响，并与 ordinary applications 分开披露 当前客户口径下单列 state-significant signals 为 1,920
NSW Spatial Services +
NSW Planning Map Services		 来自 NSW Spatial Services 的 parcel / property layers，以及新州规划与环境公开地图服务中的 zoning、FSR、height、minimum lot size、heritage、land reservation、flood planning、airport noise、OLS、biodiversity、bushfire prone land 等法定控制和风险图层。 形成 zoning、FSR、height、flood、bushfire、heritage、biodiversity、airport OLS 等 site/precinct 风险判断 进入 friction score、site-level red-flag stack 与 site screening penalty
Weekly Snapshot Archive 这是我们的内部正式归档层。它不是新的外部数据源，而是把上面这些公开来源在某一周的状态按日期冻结下来，并记录 source range、snapshot date、captured at、baseline type。 保证 Full / Delta / Bundle 不依赖“当下 latest 状态”，而依赖可复现的 dated snapshot 当前正式周快照目录：snapshots/weekly/2026-04-05/

用客户更容易理解的话说

我们不是拿一份别人整理好的二手 Excel 来做排名，而是直接从 NSW Planning Portal、NSW Spatial Services、以及新州规划与环境公开地图服务中抓取原始公开资料，再把它们统一整理成每周可比较、可解释、可复盘的开发机会与风险视图。

为什么这能体现权威性

我们的结论不是从单一榜单或人工筛选开始，而是建立在政策、申请、法定控制与空间风险等多个公开权威层之上。

为什么这能体现广泛性

同一套周报同时覆盖 precinct 级、site 级、proposal 级、activity 级与 methodology 级信息，而不是只输出一个 ranking。

为什么这能体现专业性

每次快照都写明 snapshot_date、captured_at、baseline_type 与 source range，减少“看起来专业、但不可追溯”的风险。

Sydney Council Context Coverage

`Housing target context` 在当前产品定义的 Sydney target-council scope 内覆盖 22/22 councils。这意味着 target pressure 可以作为该 scope 下的正式 context layer 使用；但这个说法不应被扩展理解成“所有通常意义上的 Greater Sydney LGA 都已经进入这一层”。

2. 数据处理流水线的条理与自动化程度

Daily 维护层 + Weekly 归档层 + Client Delivery 组装层

Daily / Source Maintenance

  1. 同步 planning proposals，更新 stage 与 stage history。
  2. 同步 ordinary applications 与 state-significant rows，写入 application_signals。
  3. 将公开数据统一落到 Supabase 分析层，供 shortlist / screening / dashboard 使用。
  4. 如需要，还可运行 coverage readiness 评估，判断扩区前的数据稳定性。

说明：连接层内置 direct / pooler fallback，支持分页、focus councils、config 继承与范围控制。

Weekly / Dated Reporting

  1. 构建 precinct shortlist 与 site screening layer。
  2. 冻结 weekly snapshot，保存 precinct、site、proposal、activity 四类 dated artifacts。
  3. 基于 snapshot 生成 Full Report、Delta Report、Methodology Appendix。
  4. 生成 Top 10 Insights、Weekly Radar、Top Site Screening、Site Cards、Deep Dives。
  5. 渲染 HTML，导出 customer bundle，并按客户阅读路径重新分组。

说明：Delta 现在默认只允许使用更早的 formal snapshot 作为 previous baseline；如需 transition delta，必须显式允许 reconstructed baseline。

Production Layer

把 dated full / delta / methodology 作为核心生产物，而不是临时从 latest view 即时拼接。

Assembly Layer

把客户包组装、HTML 渲染、bundle 导出从“分析逻辑”中拆出来，便于稳定复用和交付重组。

Snapshot Discipline

快照不可随意重写，默认要求 dated folder、manifest、source range 一致，保证可复核性。

Automation Value

自动化不是为了省人工，而是为了让每周都能重复生产、重复比较、重复解释，而不靠临场手工拼接。

3. 我们实际交付的不是“一个文件”，而是一组有逻辑层次的输出

结论层 → 变化层 → 动作层 → 证据层 → 方法层

01 核心结论

Full Report 给出当前完整盘面：Top Takeaways、Priority Precincts、Priority Site Shortlist、Risk Watchlist、Policy Pipeline、Activity Snapshot、Recommended Actions。

02 周度变化

Delta Report 解释 compared-to-previous 发生了什么：新增、减少、升降级、activity 变化、transition reliability 与高优先新目标。

03 本周动作

Top 10 Insights 和 Top Site Screening 把“大盘判断”转成“本周该跟进哪些 precinct / 哪些 lot”的短名单。

04 Supporting Evidence

Site Cards 下钻到 lot level，Precinct Deep Dives 下钻到 precinct level。它们不是重复主报告，而是为 shortlisted targets 提供可读证据链。

05 Visual Summary

Hero Visual Pack 与 Opportunity Map 提供空间感与优先级图像表达，方便更快和更高层地沟通机会与风险。

06 Methodology & Boundaries

Methodology Appendix 明确公式、阈值、比较边界、数据窗口和当前 gaps，降低“黑盒报告”的疑虑。

当前客户包的结构意义

我们已经把客户主路径重新组织为 5 步决策路径：先看本周完整结论，再看本周变化，再看本周动作，再看 supporting evidence，最后看 visual summary 与 methodology。这不是简单换目录名，而是把“内部产物视角”重排成“客户决策视角”。

4. 工作量体现在哪里

难点不在“写文字”，而在“把文字背后的系统做成每周可复用”

看得见的工作

  • 写出结构清晰、客户可读、可比较的 weekly outputs。
  • 将复杂 source layer 压缩成可以用于投资人、开发商、客户讨论的简洁结论。
  • 把 site-level 与 precinct-level evidence 做成可下钻、可导航、可打印的交付包。

更关键但不一定立刻看见的工作

  • 清洗不同公开源的时间口径、字段口径、重复问题和 stage/state 差异。
  • 把 latest-state views 改造成可冻结、可比对、可复盘的 dated snapshots。
  • 设计 formal / reconstructed baseline 规则，避免“看起来像周报，实际上不可比较”。
  • 维护评分公式、site band 阈值、风险降级规则与客户可见 methodology 的一致性。
  • 把交付包从技术目录变成客户可顺脑子阅读的 decision path。
工作单元 不是简单人工汇总的原因 体现出的价值
数据接入与更新 涉及多源公开接口、分页、focus council、state-significant 补层与结构化入库 保证覆盖面和更新频率，而不是依赖一次性人工下载
评分与筛选 需要把 policy、timing、risk、site fit、constraint penalty、title complexity 放入统一逻辑 把“信息很多”转成“优先级明确”
快照与 delta 需要处理 snapshot_date、captured_at、source ranges、formal / reconstructed baseline 与 diff identity 把“当前状态”升级成“可持续周报产品”
解释与包装 需要把技术层、分析层、交付层拆开，再按客户使用顺序重组 提升客户理解速度，也提升外部对工作量和价值的尊重

Why This Matters In The Current Pack

当前 `Top 10 Insights`、`Council Context` 和相关压力判断，建立在完整的 target-council scope 之上。因此，target pressure 可以作为这一 scope 内的正式 council-context layer 使用，但仍应与 policy、activity、risk 和 site fit 一起阅读，而不是单独决定优先级。

当前 source range provenance

  • Planning proposals: 2026-04-01 至 2026-04-02
  • Application signals: 2022-07-07 至 2026-04-02
  • Site screening layers: 2026-04-04 至 2026-04-05

当前交付中最能体现价值的内容层次

  • Full Report 内部同时包含 precinct priority、site shortlist、risk watchlist、policy pipeline、activity snapshot 与 weekly actions。
  • Delta Report 将“这周相对上周的变化”单独成层，而不是让客户自己从 cumulative numbers 里猜。
  • Methodology Appendix 把公式、阈值、可比性边界和 gaps 公开出来，提升可解释性和可信度。

6. 一句话总结

为什么值得尊重这项工作的工作量与价值

我们交付的不是“一个报告文件”，而是一个把多源公开权威数据、自动化处理流水线、可解释评分逻辑、周快照治理、客户化交付结构和可持续复用能力整合在一起的研究与产品系统。
